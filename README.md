# E1-1 iMac 기반 AI/SW 개발 워크스테이션 구축

## 프로젝트 개요
이 과제는 iMac(macOS) 환경에서 개발을 시작할 수 있도록 기본 작업셋팅을 점검하고 기록하는 것을 목표로 한다.
터미널 사용, 파일 및 디렉토리 권한, Docker 기반 컨테이너 실행, Git/GitHub 연동까지 직접 확인하여 특정 PC에만 의존하지 않는 재현 가능한 개발 환경을 만든다.

이 문서는 `README.md` 초안이며, 실제 실습 결과에 따라 계속 보완한다.

## 과제 목표
- iMac 환경에서 개발에 필요한 기본 도구를 설치하고 동작을 확인한다.
- 터미널 명령과 권한 변경 과정을 기록한다.
- Docker 이미지/컨테이너 실행, 포트 매핑, 데이터 영속성을 검증한다.
- Git 및 GitHub 연동 상태를 점검하고 결과를 문서화한다.

## 수행 범위

### 1. 터미널 및 권한 확인
- 현재 위치 확인: `pwd`
- 파일/디렉토리 목록 확인: `ls -la`
- 디렉토리 이동: `cd`
- 디렉토리 생성: `mkdir`
- 파일 생성: `touch`
- 파일 내용 확인: `cat`
- 복사/이동/이름 변경: `cp`, `mv`
- 삭제: `rm`
- 권한 변경: `chmod`

기록해야 할 내용:
- 파일 1개에 대한 권한 변경 전/후 결과
- 디렉토리 1개에 대한 권한 변경 전/후 결과
- 명령 실행 과정과 확인 결과

### 2. Docker 기본 동작 확인
- 설치 상태 확인: `docker --version`
- Docker 엔진 정보 확인: `docker info`
- 테스트 컨테이너 실행: `docker run hello-world`
- Ubuntu 컨테이너 진입: `docker run -it ubuntu bash`
- 이미지 목록 확인: `docker images`
- 컨테이너 목록 확인: `docker ps -a`
- 로그 확인: `docker logs`
- 자원 사용량 확인: `docker stats`

### 3. Dockerfile, 포트, 스토리지 검증
- Dockerfile을 사용해 커스텀 이미지 1개 생성
- 컨테이너 실행 시 `-p` 옵션으로 포트 매핑 확인
- 브라우저 접속 또는 `curl`로 응답 확인
- 바인드 마운트 또는 Docker 볼륨으로 데이터 영속성 검증

### 4. Git 및 GitHub 연동
- Git 설정 확인: `git config --list`
- 로컬 저장소 상태 확인: `git status`
- 원격 저장소 연결 확인: `git remote -v`
- 커밋 및 푸시 기록
- GitHub 저장소에서 결과물 확인

## 보안 및 작성 원칙
- 문서와 로그에는 토큰, 비밀번호, 개인 이메일, 계정 비밀값을 그대로 기록하지 않는다.
- 필요한 경우 민감 정보는 마스킹 처리한다.
- 실제 결과와 캡처를 기준으로 작성하고, 추정 내용은 별도로 표시한다.
